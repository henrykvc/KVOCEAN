# 🌊 KV OCEAN 온보딩 가이드

> 💡 **KV OCEAN은?** 피투자사 분기 재무제표를 검증하고, 쌓고, 지표로 만드는 카카오벤처스 내부 도구입니다. 이 문서 하나로 처음 쓰는 분도 바로 시작할 수 있게 만들었습니다.

---

## 1. 한눈에 보기

KV OCEAN이 하는 일은 딱 세 가지입니다.

1. **검증** — 로제봇(OCR)이 읽어 온 재무제표 숫자를 붙여넣으면, 합계가 맞는지·계정이 분류돼 있는지 자동으로 검사합니다.
2. **누적** — 검사를 100% 통과한 데이터만 회사·분기별로 저장합니다.
3. **결과물** — 저장된 데이터로 런웨이 · EBITDA · 월 평균 지출액 같은 지표를 계산하고, 구글시트에 자동으로 보내줍니다.

**전체 흐름 (왼쪽에서 오른쪽으로)**

> 📄 재무제표 PDF/이미지 → 🤖 로제봇이 OCR → 📊 구글시트에 "3줄 텍스트"로 쌓임 → ✅ **KV OCEAN**에 붙여넣어 검증·저장 → 📈 결과물 지표 + 구글시트 자동 동기화

> ⚠️ **OCR(글자 추출) 자체는 KV OCEAN이 하지 않습니다.** 로제봇이 떠준 텍스트를 받아서 검증·누적하는 도구입니다. 로제봇 사용법은 별도의 OCR 매뉴얼을 참고하세요.

---

## 2. 꼭 알아야 할 개념 3가지

처음엔 이 세 가지만 알면 됩니다.

### ① 3줄 텍스트

로제봇이 만들어 주는 표준 입력 형식입니다. 그대로 복사해서 붙여넣으면 됩니다.

| 행 | 내용 | 예시 |
| --- | --- | --- |
| 1행 | 섹션(분류) | 자산, 부채, 손익… |
| 2행 | 계정명 | 현금및현금성자산, 매출액… |
| 3행~ | 분기별 값 | 1,234,567 |

### ② 계정트리 (분류DB)

모든 계정명을 분류하는 **사전**입니다. 구글시트 `KVocean_계정트리_v2`가 유일한 원본이고, 여기에 없는 계정은 **"미분류"**로 표시됩니다. 미분류 계정은 결과물 계산에서 빠지기 때문에 관리자가 분류해 줘야 합니다.

### ③ 덮어쓰기 원칙

같은 회사·같은 분기를 다시 저장하면 **이전 저장본을 덮어씁니다** (누적이 아님). 수정하고 다시 저장하는 게 기본 사용법입니다.

## 3. 시작하기

### 로그인

1. 배포 URL 접속 → **Google로 로그인**
2. 초대된(화이트리스트 등록) 계정만 들어올 수 있습니다.
3. 초대가 안 된 계정은 로그인 화면에서 **「접근 요청 보내기」**로 사유를 적어 제출하면, 관리자가 승인해 줍니다.

### 권한 (3단계)

| 역할 | 누구 | 할 수 있는 일 |
| --- | --- | --- |
| 매니저 | 일상 사용자 | OCR검증 · 데이터 · 결과물 · 휴지통 조회 |
| 관리자 | 운영 담당 | 위 전부 + 검증 규칙 · 수식 · 분류DB · 계정 초대(/admin) |
| 제작자 | Henry | 위 전부 + 데이터 삭제 · 휴지통 복구/완전삭제 |

> 🔒 사이드바에서 자물쇠가 붙은 탭은 권한이 부족한 것입니다 (관리자 이상 필요).

### 화면 구조

상단에서 두 가지 뷰를 전환합니다.

- **작업 메뉴** — 왼쪽 사이드바에 탭 7개. 일상 업무는 전부 여기서.
- **최종결과물** — 회사·분기 4개를 가로로 놓고 비교하는 보기 전용 뷰.

| 탭 | 이름 | 어느 가이드에서 다루나 | 권한 |
| --- | --- | --- | --- |
| 1 | OCR검증 | 4. 데이터 입력 가이드 | 전원 |
| 1-1 | 검증 규칙관리 | 7. 분류DB 관리 가이드 | 관리자 |
| 2 | 데이터 | 5. 데이터 관리 가이드 | 전원 |
| 3 | 결과물 | 6. 결과물 가이드 | 전원 |
| 3-1 | 수식 | 6. 결과물 가이드 | 관리자 |
| 4 | 분류DB | 7. 분류DB 관리 가이드 | 관리자 |
| 🗑️ | 휴지통 | 5. 데이터 관리 가이드 | 조회 전원 / 복구는 제작자 |

> 📝 **워크스페이스 메모** — 사이드바 맨 아래에 팀 공유 메모장이 있습니다. 입력하면 자동 저장되고, 누가 언제 고쳤는지 표시됩니다.

## 4. 데이터 입력 가이드

> 🎯 **목표**: 로제봇이 떠준 3줄 텍스트를 검증해서 통과율 100%로 만들고 저장하기. 사용하는 탭: **1. OCR검증**

### 기본 순서

1. 입력 패널의 `구글시트 ↗`를 눌러 3줄 데이터 시트를 엽니다.
2. 해당 회사의 3줄 텍스트를 복사해서 **「3행 OCR 텍스트」 칸에 붙여넣기** → 즉시 자동 검증이 돌아갑니다.
3. **회사명**을 입력합니다. (저장 시 회사를 구분하는 기준)
4. 상단 카드에서 **통과율**과 **미분류 계정** 수를 확인합니다.
5. 실패·미분류가 있으면 아래 방법으로 정리합니다.
6. **별도/연결**을 선택하고 `저장하기` → 구글시트 동기화까지 자동으로 끝납니다.

> ✅ `저장하기`는 **통과율 100% + 검증 항목 1건 이상**일 때만 켜집니다. 회색이면 아직 실패 항목이 남아 있다는 뜻입니다. 같은 회사·분기가 이미 있으면 덮어쓰기 확인 후 저장됩니다.

### 입력 패널 (왼쪽)

| 요소 | 설명 |
| --- | --- |
| 허용 오차 (원) | 합계 검증에서 이 금액 이하 차이는 통과로 인정 (기본 1원) |
| 회사명 | 저장 시 회사 구분 키. 회사별 규칙이 있으면 "회사 규칙 적용 중" 태그 표시 |
| 3행 OCR 텍스트 | 붙여넣으면 즉시 검증 시작 |
| `입력 수정 초기화` | 이번 세션에서 한 값·계정명·부호 수정을 전부 리셋 |
| `구글시트 ↗` | 3줄 데이터 원본 시트를 새 탭으로 열기 |

### 실패 항목 고치기

`실패만 펼치기`로 실패한 검증 카드를 열면 다음 도구가 있습니다.

| 도구 | 설명 |
| --- | --- |
| 원인 추정 | 차이를 0원으로 만드는 원인 후보를 자동 제시. **가장 먼저 확인하세요** — 추정 정보만 제공하고, 수정은 OCR 수정값·계정명에서 직접 |
| OCR 수정값 입력 | 셀 값을 직접 고치면 **수정한 값이 그대로 저장**됩니다 |
| 검증 부호 선택 | 가산(+) / 차감(−). **미분류 계정에서만 선택 가능** — 계정트리에 분류된 계정은 🔒 잠금이고 부호는 분류DB(계정트리 시트)에서만 변경 (7장) |
| 🗑 계정 삭제 | 해당 계정 열을 데이터에서 제거 |
| `행 추가` | 누락된 계정을 직접 추가 |

### 미분류·오타 정리하기

중앙 미리보기 표에서 계정명 셀은 **직접 수정 가능**합니다 (회사명·날짜 칸은 잠김). 미분류 계정 밑에는 도우미 칩이 붙습니다.

| 칩 | 의미 |
| --- | --- |
| 🔤 오타? `후보명` | 클릭하면 그 이름으로 교체. 과거 분기에서 쓰던 비슷한 계정명을 찾아 제안 |
| 🆕 미분류 · 신규일 수 있음 | 비슷한 이름이 없는 진짜 새 계정 |
| ➕ 신규 유지 | "오타 아님, 새 계정 맞음"이라고 확정 (칩 끄기) |

> 💡 오타가 아닌 **진짜 신규 계정**은 그대로 저장해도 됩니다. 다만 결과물 계산에서 빠지므로, 관리자에게 분류를 요청하세요 (7장의 미분류 처리 사이클).

### OCR 정리본 (오른쪽 패널)

수정이 반영된 최종본을 섹션별 표로 확인할 수 있습니다. 행 추가/삭제 가능, `클립보드 복사`로 텍스트 복사.

## 5. 데이터 관리 가이드

> 🎯 **목표**: 저장된 데이터를 찾고, 속성을 고치고, 빠진 회사를 챙기기. 사용하는 탭: **2. 데이터**, **🗑️ 휴지통**

### 찾기 · 확인하기

저장된 데이터는 **회사별 카드**로 쌓입니다.

- **검색/필터 바** — 회사명 검색 + 분기 필터 + 산업 필터(미분류 포함) + `초기화`
- **분기 칩** — 카드 안에서 클릭으로 분기 선택/해제. 연결 재무제표는 "-연결" 접미사가 붙음
- `결과물 보기` — 선택한 분기를 들고 결과물 탭으로 이동

### 패밀리 수집 현황 챙기기

검색 바 옆의 **패밀리 N / M 칩**은 "이번 분기 패밀리사 M곳 중 N곳 저장 완료"라는 뜻입니다.

1. 칩을 클릭하면 **미저장 회사 명단**이 펼쳐집니다.
2. `📋 미저장 명단 복사`로 복사해서 후속 처리(자료 요청 등)에 쓰세요.
3. 분기 필터를 바꾸면 그 분기 기준으로 다시 계산됩니다. "전체 분기"면 최신 분기 기준.

### 수정하기

- **✏️ 편집 모드**를 켜면 산업(서비스/게임/기술/헬스케어/크립토), 회계기준(K-GAAP/IFRS), 선택 분기의 별도/연결을 수정할 수 있습니다.
- **숫자·계정을 고치려면** → `검증기로 불러오기`로 OCR검증 탭에 다시 로드 → 수정 → 재저장 (덮어쓰기).

### 삭제 · 복구하기

- `삭제` 버튼은 **제작자만** 보입니다. 삭제하면 완전히 지워지는 게 아니라 **휴지통**으로 이동합니다.
- 휴지통은 누구나 조회할 수 있고, `복구하기` / `완전삭제`는 제작자만 가능합니다. 필요하면 Henry에게 요청하세요.

## 6. 결과물 가이드

> 🎯 **목표**: 저장 데이터로 계산된 지표를 확인하고 구글시트로 보내기. 사용하는 탭: **3. 결과물**, **3-1. 수식**, 상단 **최종결과물** 뷰

### 지표 확인하기

- **지표 표** — 섹션별 항목 × 분기 (금액 / 비율 / 전분기 증감율)
- **계산 검증** — 지표를 펼치면 공식·입력값·세부 구성까지 근거가 전부 보입니다. "이 숫자 어디서 나왔지?" 싶을 때 펼쳐 보세요.
- **수식 탭(3-1, 관리자)** — 지표 43개의 계산식을 읽기 전용으로 열람 (예: 유동비율 = 유동자산 ÷ 유동부채 × 100). 수식 자체는 여기서 편집할 수 없습니다.

### 여러 회사·분기 비교하기

상단에서 **최종결과물** 뷰로 전환하면 슬롯 4개에 회사·분기를 골라 가로로 비교할 수 있습니다. `동일 회사` 토글을 켜면 슬롯1 회사의 연속 4개 분기가 자동 선택됩니다.

### 구글시트로 보내기

| 방법 | 언제 |
| --- | --- |
| 자동 동기화 | OCR검증에서 `저장하기`를 누를 때마다 그 회사·분기가 자동으로 시트에 반영됨 — 따로 할 일 없음 |
| `전체 회사 시트 동기화` | 저장된 **모든** 회사·분기를 한 번에 다시 전송. 분류DB나 수식이 바뀐 뒤 전체를 갱신할 때 사용 |

- 시트에는 분기별 탭(YYMM)에 **KV ID · 회사명 · 런웨이 · EBITDA · 월 평균 지출액**이 기록됩니다.
- `구글시트 ↗`로 결과물 시트를 바로 열 수 있고, 동기화 중/완료/실패 상태가 화면에 표시됩니다.

## 7. 분류DB 관리 가이드 (관리자)

> 🎯 **목표**: 계정트리를 관리해서 미분류를 없애고, 분류·부호 변경이 데이터에 잘 반영되게 하기. 사용하는 탭: **4. 분류DB**, **1-1. 검증 규칙관리**

> ⚠️ **대원칙: 분류·부호의 원본은 구글시트 `KVocean_계정트리_v2`(탭 "통합") 하나뿐입니다.** 분류DB 탭은 그 시트를 비추는 읽기 전용 거울이라, 수정은 항상 시트에서 하고 「시트에서 동기화」로 반영합니다.

### 미분류 처리 사이클 (가장 자주 하는 일)

1. `미분류` 보기로 전환 → 트리에 없는 계정이 빨간색으로 나열됨
2. `미분류 N건 시트에 추가` → 구글시트에 '분류 대기' 행으로 일괄 추가됨 (중복은 건너뜀)
3. 구글시트에서 분류·코드·부호를 채움
4. 돌아와서 `시트에서 동기화`
5. (필요시) `점검 실행`으로 기존 저장 데이터에 미친 영향 확인

### 분류DB 탭 버튼 정리

| 버튼 | 설명 |
| --- | --- |
| `시트에서 동기화` | 시트를 읽어 검증 후 반영. **오류가 1건이라도 있으면 전체 거부**하고 직전 정상본 유지 |
| `점검 실행` | 현재 분류·규칙으로 모든 저장 데이터를 재검증 → 실패 목록 표시. `검증기로 열기`로 바로 수정 가능 |
| `분류표`/`미분류` 전환 | 전체 트리 보기 ↔ 미분류 계정만 보기 |
| `미분류 N건 시트에 추가` | 미분류를 시트에 분류 대기 행으로 추가 |
| 미분류 행 삭제 | OCR 오류 등으로 잘못 들어온 미분류 계정을 저장 데이터에서 바로 제거 (원문에서 그 열을 빼고 해당 분기를 재저장) |
| 출처 칩 | 그 계정이 등장한 회사·분기. 클릭하면 OCR검증 탭에 해당 데이터가 열림 |

> 💡 **"시트에 분명히 추가했는데 동기화해도 안 들어와요"** — 1순위 원인은 시트 어딘가의 검증 오류입니다. 동기화는 all-or-nothing이라 오류 1건만 있어도 전체가 거부됩니다. 동기화 결과 메시지의 오류 건수를 확인하고 시트를 고치세요.

### 부호(±) 영구 변경하기

- OCR은 +/− 기호를 못 읽기 때문에 값은 절대값으로 받고, **더할지 뺄지는 계정트리의 부호 열이 결정**합니다.
- 계정트리에 분류된 계정은 OCR검증 카드에서 부호가 **🔒 잠겨 있습니다**. 변경은 한 곳에서만: 계정트리 시트에서 그 계정의 부호 수정 → `시트에서 동기화`.
- 미분류 계정(트리에 없는 것)만 검증 카드에서 부호를 정할 수 있고, 이건 **그 세션 전용**입니다 — 트리에 분류되는 순간 트리 부호가 적용됩니다.
- 같은 계정명인데 케이스마다 부호가 달라야 하면 **계정명을 분리**해서 해결하세요 (예: `_minus`/`_plus` 접미사, 환입/전입 분리).

### 자본 검증 보조 규칙 (1-1 탭)

자본 합계 검증에만 쓰이는 보조 규칙입니다. (계정 매칭·부호는 전부 계정트리 소관)

- **자본 구성항목 규칙** — 계정명 + 가산/차감 + 상위 항목
- **자본 검증 제외 항목** — 당기순이익처럼 메모성 항목 제외
- **부모 항목 별칭** — "자본 = 자본총계 = 총자본" 같은 동의어
- `되돌리기`(직전 편집 취소) / `편집값 반영`(저장 — 전 사용자 공유)

## 9. 분기 운영 루틴 (운영자)

분기 마감 시즌에 운영자가 도는 순서입니다. 각 단계의 자세한 방법은 해당 가이드 참고.

1. 로제봇 OCR → 3줄 데이터가 OCR검토용 시트에 누적
2. **2차 검증 배치** 실행 — 원본 재무제표 파일(Drive)과 대조해 깨진 계정명·값을 자동 보정 → 시트에 `코드연결용시트_보정` + `보정내역` 탭 생성
3. 보정 탭의 데이터를 OCR검증에 붙여넣어 **사람이 최종 검증** → 저장 (→ 4장)
4. 데이터 탭 **패밀리 N/M 칩**으로 빠진 회사 확인 → 미저장 명단 복사해서 후속 처리 (→ 5장)
5. 결과물 확인 → `전체 회사 시트 동기화` (→ 6장)
6. 미분류 계정은 분류DB의 미분류 처리 사이클로 정리 (→ 7장)

## 11. 용어 사전

| 용어 | 뜻 |
| --- | --- |
| 로제봇 | 재무제표 PDF/이미지에서 글자를 떠주는 외부 OCR 도구. KV OCEAN의 입력 소스 |
| 3줄 텍스트 | 섹션 1줄 + 계정명 1줄 + 분기 값 1줄(이상)의 표준 입력 형식 |
| 계정트리 / 분류DB | 구글시트로 관리하는 5단계 계정 분류 체계. 계정 매칭과 부호의 유일한 원본 |
| 미분류 | 계정트리에 없는 계정명. 결과물 계산에서 빠지므로 분류가 필요함 |
| 부호 | 계정별 가산(+)/차감(−) 설정. OCR이 +/− 기호를 못 읽기 때문에 값은 절대값으로 받고, 더할지 뺄지는 계정트리가 결정 |
| 세션 부호 수정 | **미분류 계정**의 부호를 검증 카드에서 임시로 정하는 것. 이번 검증에만 적용. 계정트리에 분류된 계정은 부호가 잠겨 있어(🔒) 세션 수정 불가 |
| 별도/연결 | 재무제표 유형. 저장 시 선택, 데이터 탭에서 변경 가능 |
| 덮어쓰기 | 같은 회사·같은 분기 재저장 시 이전 저장본을 교체하는 정책 |
| 휴지통 (soft delete) | 삭제해도 실제로 지우지 않고 표시만 함. 제작자가 복구하거나 완전삭제 |
| 정합성 점검 | 현재 분류·규칙으로 모든 저장 데이터를 재검증하는 기능 (분류DB 탭) |
| 2차 검증 배치 | 분기마다 원본 재무제표와 대조해 OCR 데이터의 계정명·값을 자동 보정하는 운영 스크립트 |
| 패밀리 N/M | 데이터 탭 칩. 이번 분기 패밀리사 M곳 중 N곳 저장 완료라는 뜻 |
| KV ID | 결과물 시트에서 회사를 식별하는 코드 (KV_000 형식) |

### 검증 엔진 동작 순서

1. 3줄 텍스트 파싱 (섹션 / 계정명 / 값)
2. 계정 매칭 — 계정명 정규화 후 계정트리에서 검색, 섹션 힌트로 오매칭 방지
3. 부호 적용 — 트리의 부호로 가산/차감 결정 (세션 수정이 있으면 우선)
4. 합계 검증 — 허용 오차 안이면 통과
5. 실패 시 진단 — 차이를 0원으로 만드는 수정 후보 자동 탐색

**합계 검증 규칙**: 자산 = 유동자산 + 비유동자산 / 자산 = 부채 + 자본 / 부채 = 유동부채 + 비유동부채 / 영업이익 = 매출액 − 매출원가 − 판관비 / 법인세차감전이익 · 당기순이익 연결 / 매출원가 구성 / 자본 구성(1-1 규칙 기반) / 각 섹션 소계

### 데이터 보존 원칙

- **수정 반영본을 그대로 저장** — 저장 시점의 값·계정명 수정이 반영된 텍스트가 원문이 됨. 검증기로 다시 불러오면 저장 당시 데이터가 그대로 열림 (옛 저장본의 수정 레이어는 읽기 호환 유지)
- **계정트리 캐시는 all-or-nothing** — 시트 검증 오류가 1건이라도 있으면 갱신하지 않고 직전 정상본 유지
- **삭제는 2단계** — 데이터 탭 삭제 = 휴지통 이동(표시만) → 휴지통의 완전삭제만 실제 제거 (제작자 전용)

### 계정트리 코드 체계

13자리 코드 = 대분류(1) + 중분류(3) + 소분류(3) + 세분류(3) + 계정(3). 동기화 시 코드 형식·중복·계층 깊이·부모 존재·말단 부호 필수 여부를 검사합니다.
